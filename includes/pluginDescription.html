SkillsJars Helper is a JetBrains IDE plugin that brings discovery, preview, export and publishing workflows for Agent Skills distributed through SkillsJars directly into your IDE.
SkillsJars Helper 是一款 JetBrains IDE 插件, 直接在 IDE 内提供通过 SkillsJars 分发的 Agent Skills 的发现、预览、导出与发布工作流.

GitHub | SkillsJars

Modern AI-assisted development is increasingly powered by reusable Agent Skills—directories that contain a SKILL.md plus optional scripts and assets. SkillsJars lets you ship those skills as Maven / Gradle artifacts. SkillsJars Helper turns those artifacts into first-class IDE citizens: every classpath JAR carrying skills shows up in a dedicated tool window, you can read SKILL.md without unpacking the JAR, and (in upcoming milestones) you will be able to export skills into local Agent directories such as .claude/skills, .codex/skills, and .junie/skills with one click.

Core Capabilities

  • Dependency scanning: discovers JARs that contain META-INF/skills/**/SKILL.md or META-INF/resources/skills/**/SKILL.md across Maven regular dependencies and Maven plugin <dependencies>.
  • SkillsJar preview: opens skill frontmatter and Markdown content directly from inside JARs — no temporary extraction required.
  • Tree-based tool window: groups skills by artifact, ships a dedicated description panel, and supports speed-search, tooltips, double-click / Enter / right-click menus.
  • Open extension point: third-party plugins can register custom SkillSourceScanner implementations to support Gradle, SBT or proprietary build systems.
  • Agent export (planned): prepares extraction to local Agent directories such as .claude/skills, .codex/skills, and .junie/skills with manifest bookkeeping.
  • Publishing preparation (planned): validates a local skill set before packaging and publishing it as a SkillsJar.

Development Status

  • Phase 1 — Available: SkillsJars discovery (Maven), SKILL.md parsing, public SkillRegistry API, and the tree-based Agent Skills tool window.
  • Phase 2 — Planned: skill export to local Agent directories, manifest writing, and pre-publish validation.
  • Phase 2 — Planned: Gradle dependency scanner, plugged in through the same skillSourceScanner extension point.

现代 AI 辅助研发越来越依赖可复用的 Agent Skills — 一个目录, 包含 SKILL.md 与可选的脚本/资源. SkillsJars 把这些 skill 以 Maven / Gradle artifact 的形式分发. SkillsJars Helper 让这类 artifact 在 IDE 中获得一等公民待遇: classpath 中所有携带 skill 的 JAR 都会在专用工具窗里列出, 你可以 不解压直接预览 SKILL.md; 在后续版本中, 你将能够一键把 skill 导出到本地 Agent 目录, 例如 .claude/skills, .codex/skills, .junie/skills.

核心能力

  • 依赖扫描: 发现 Maven 普通依赖与 Maven plugin <dependencies> 中包含 META-INF/skills/**/SKILL.md 或 META-INF/resources/skills/**/SKILL.md 的 JAR.
  • SkillsJar 预览: 直接读取 JAR 内的 frontmatter 与 Markdown 正文, 无需解压到临时目录.
  • 树形工具窗: 按 artifact 分组、独立的描述面板, 支持 SpeedSearch / Tooltip / 双击 / 回车 / 右键菜单.
  • 开放扩展点: 三方插件可实现 SkillSourceScanner, 接入 Gradle / SBT 或自有构建系统.
  • 导出到 Agent (规划中): 一键导出 skill 到本地 Agent 目录 (例如 .claude/skills / .codex/skills / .junie/skills) 并维护 manifest.
  • 发布前检查 (规划中): 在打包发布 SkillsJar 前对本地 skill 进行结构校验.

开发状态

  • 一期 — 已就绪: SkillsJars 解析 (Maven), SKILL.md 解析, 公共 SkillRegistry API, 以及树形 Agent Skills 工具窗.
  • 二期 — 规划中: Skill 导出到本地 Agent 目录, manifest 写入, 发布前结构校验.
  • 二期 — 规划中: Gradle 依赖扫描, 通过同一 skillSourceScanner 扩展点接入.
