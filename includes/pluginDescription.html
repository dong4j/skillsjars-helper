SkillsJars Helper is a JetBrains IDE plugin that makes Agent Skills shipped as Maven artifacts (SkillsJars) first-class citizens: discover them on the classpath, preview SKILL.md straight from inside the JAR, and export any skill to a local Agent directory with one click.

Why

Agent Skills — a SKILL.md plus optional scripts and assets — are how modern coding agents (Claude Code, Codex, Junie, Cursor, Gemini, Qoder, Trae, CodeBuddy, …) pick up reusable behaviour. SkillsJars packages them as plain Maven / Gradle artifacts so a team can version and share skills like any other dependency. This plugin closes the last gap: making those skills visible and usable inside the IDE without ever unpacking a JAR.

Features

  • Discovery: scans Maven regular dependencies and Maven plugin dependencies for META-INF/skills/**/SKILL.md and META-INF/resources/skills/**/SKILL.md; exposes results through a public project-level SkillRegistry service.
  • Agent Skills tool window: skills are grouped by artifact in a tree, with a dedicated description panel below a draggable splitter. Double-click or Enter opens SKILL.md in-place; right-click for Open / Copy Name / Copy Coordinate / Open Source Jar.
  • One-click export: export a skill (or the whole JAR) into nine preset Agent directories — .claude/skills, .codex/skills, .junie/skills, .agents/skills, .cursor/skills, .gemini/skills, .qoder/skills, .trae/skills, .codebuddy/skills — or a custom directory. Each exported folder gets a manifest with per-file sha256, so upgrades, local edits and same-name conflicts are detected automatically.
  • Install state badges: every skill leaf right-aligns brand icons of the Agent directories it is already installed into; tooltip spells out the agent ids in plain text.
  • Open extension point: skillSourceScanner lets third-party plugins add Gradle, SBT or proprietary build systems without touching the coordinator.

How to use

  1. Add any SkillsJars to your project (as a regular Maven dependency or under skillsjars-maven-plugin's ＜dependencies＞).
  2. Open the Agent Skills tool window on the right; the skills will appear, grouped by artifact.
  3. Double-click a skill to read its SKILL.md; right-click to export it to the Agent directory of your choice. The badges on each skill leaf show where it is already installed.

SkillsJars Helper 让以 Maven artifact 形式分发的 Agent Skills (即 SkillsJars) 在 JetBrains IDE 中 成为一等公民: 在 classpath 上自动发现、不解压直接预览 SKILL.md, 并一键导出到本地 Agent 目录.

这是什么

Agent Skills (一份 SKILL.md 加可选脚本 / 资源) 是现代 Coding Agent (Claude Code、Codex、Junie、 Cursor、Gemini、Qoder、Trae、CodeBuddy …) 复用行为的标准载体. SkillsJars 把它们打成 Maven / Gradle artifact, 让团队像管理普通依赖一样版本化和共享 skill. 本插件补上 IDE 里的最后一公里: 不解压、不离开 IDE, 即可发现、预览和分发这些 skill.

主要能力

  • 依赖扫描: 自动扫描 Maven 普通依赖与 Maven plugin dependencies, 解析 JAR 内 META-INF/skills/**/SKILL.md 与 META-INF/resources/skills/**/SKILL.md, 通过项目级 SkillRegistry 服务对外暴露.
  • Agent Skills 工具窗: 按 artifact 分组的树, 下方独立描述面板 (可上下拖动). 双击或 Enter 直接打开 SKILL.md; 右键菜单含 Open / Copy Name / Copy Coordinate / Open Source Jar.
  • 一键导出: 右键 skill (或整个 JAR) 导出到 9 个预设 Agent 目录 (.claude/skills, .codex/skills, .junie/skills, .agents/skills, .cursor/skills, .gemini/skills, .qoder/skills, .trae/skills, .codebuddy/skills) 或自定义目录. 导出时写入 manifest (含来源 artifact 与每文件 sha256), 自动判断升级、本地修改与同名冲突.
  • 安装徽标: skill 叶子节点右对齐显示已安装到的 Agent 品牌图标, tooltip 明确列出 agent id.
  • 开放扩展点: skillSourceScanner 允许三方插件扩展 Gradle / SBT 或自有构建系统的扫描, 无需改动本插件协调层.

怎么用

  1. 把任意 SkillsJars 加入项目 (作为普通 Maven 依赖, 或放在 skillsjars-maven-plugin 的 dependencies 下).
  2. 打开右侧的 Agent Skills 工具窗, 即可看到按 artifact 分组的 skill 列表.
  3. 双击 skill 查看 SKILL.md; 右键导出到目标 Agent 目录. skill 叶子节点上的徽标会实时显示当前已安装位置.
