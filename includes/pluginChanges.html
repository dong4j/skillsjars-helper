2026.1.1000 — Initial Release

  • Skill discovery: scans Maven regular dependencies and Maven plugin ＜dependencies＞, parses META-INF/skills/**/SKILL.md and META-INF/resources/skills/**/SKILL.md from inside JARs, and exposes results through a project-level SkillRegistry service.
  • Agent Skills tool window: artifacts grouped one level deep, skills as leaves; description panel below a draggable splitter; speed search, tooltips, double-click / Enter to open SKILL.md; right-click for Open / Copy Name / Copy Coordinate / Open Source Jar.
  • One-click export: export a skill (or the whole JAR) into nine preset Agent directories (.claude/skills, .codex/skills, .junie/skills, .agents/skills, .cursor/skills, .gemini/skills, .qoder/skills, .trae/skills, .codebuddy/skills) or a custom directory. Folder name follows the SKILL.md name field per the Agent Skills convention.
  • Manifest and install states: each exported folder writes a .skillsjars-helper.json with source artifact and per-file sha256, so upgrades, local edits and same-name conflicts are detected automatically. Common paths write silently with a notification; sensitive ones prompt for confirmation.
  • Install state badges: skill leaves right-align brand icons for every Agent directory the skill is already installed into; an InstallationRegistryService rescans the preset directories on startup and after every export.
  • Public extension point: skillSourceScanner lets third-party plugins add Gradle, SBT or proprietary scanners without touching the coordinator.
  • i18n: English and Simplified Chinese message bundles.
  • Skill 发现: 扫描 Maven 普通依赖与 Maven plugin ＜dependencies＞, 解析 JAR 内 META-INF/skills/**/SKILL.md 与 META-INF/resources/skills/**/SKILL.md, 通过项目级 SkillRegistry 服务对外暴露.
  • Agent Skills 工具窗: 按 artifact 一层折叠, skill 作为叶子节点; 下方独立描述面板 (上下可拖动); SpeedSearch / tooltip / 双击或 Enter 打开 SKILL.md; 右键菜单含 Open / Copy Name / Copy Coordinate / Open Source Jar.
  • 一键导出: 右键 skill (或整个 JAR) 导出到 9 个预设 Agent 目录 (.claude/skills, .codex/skills, .junie/skills, .agents/skills, .cursor/skills, .gemini/skills, .qoder/skills, .trae/skills, .codebuddy/skills) 或自定义目录; 目录名严格采用 SKILL.md 的 name 字段, 与 Agent Skills 规范一致.
  • Manifest 与安装状态: 每个导出目录写入 .skillsjars-helper.json (含来源 artifact 与每文件 sha256), 自动判断升级 / 本地修改 / 同名冲突; 常用路径静默写入 + Notification, 敏感场景弹出二次确认.
  • 安装徽标: skill 叶子节点右对齐显示已安装到的 Agent 品牌图标; InstallationRegistryService 在启动和每次导出后自动重扫所有预设目录.
  • 开放扩展点: skillSourceScanner 允许三方插件扩展 Gradle / SBT 或自有构建系统的扫描, 无需改动协调层.
  • 国际化: 中英 Bundle 同步.
