2026.1.2000

  • Skill export: a public project-level SkillExportService mirrors SkillRegistry; right-click any skill to export it (or its whole jar) to one of seven preset Agent skills directories (.claude/skills, .codex/skills, .junie/skills, .agents/skills, .cursor/skills, .gemini/skills, .qoder/skills) or a custom directory.
  • Naming aligned with Agent Skills spec: the exported folder is named after the SKILL.md frontmatter name, not the flattened skillsjars__org__repo__ form — matching the convention adopted by useSkillsNameAsDirectory=true in skillsjars-maven-plugin v0.0.7.
  • Six-state install machine: NEW / UP_TO_DATE / OUTDATED / LOCALLY_MODIFIED / FOREIGN / DUPLICATE_NAME. Frequent paths (NEW / OUTDATED) write silently with a notification; sensitive ones (LOCALLY_MODIFIED / FOREIGN) prompt yes/no; same-name conflicts from a different artifact offer a three-way choice (overwrite original / use suffixed name / cancel).
  • Manifest .skillsjars-helper.json: written into every exported skill folder with artifact + skillRoot + per-file sha256; used to detect upgrades and local edits without a heavy index.
  • Installation badges: skill leaves show a row of official brand icons (Claude, Codex, Junie, Cursor, Gemini, Qoder, plus a folder icon for custom directories) right-aligned on the same row as the skill name — the cell is stretched to the tree width so badges line up across rows even when names have different lengths. Brand assets come from @lobehub/icons (MIT). Tooltip still spells out the agent ids in plain text. An InstallationRegistryService scans the seven preset directories on startup and after every export.
  • Tests: ExportNamingTest, ManifestJsonTest, ExportPlannerTest covering all six install states; SkillJarParserTest extended to verify file enumeration.
  • Skill 导出: 新增项目级 SkillExportService, 与 SkillRegistry 对称暴露; 右键任意 skill 可导出到 7 个预设 Agent 目录 (.claude/skills, .codex/skills, .junie/skills, .agents/skills, .cursor/skills, .gemini/skills, .qoder/skills) 或自定义目录, 整个 jar 也可一键批量导出.
  • 命名对齐 Agent Skills 规范: 导出目录直接采用 SKILL.md frontmatter 的 name, 不再使用 skillsjars__org__repo__ 扁平化前缀, 与 skillsjars-maven-plugin v0.0.7 的 useSkillsNameAsDirectory=true 保持一致.
  • 6 状态写入机: NEW / UP_TO_DATE / OUTDATED / LOCALLY_MODIFIED / FOREIGN / DUPLICATE_NAME. 常用路径 (NEW / OUTDATED) 直接写入 + Notification; 敏感场景 (LOCALLY_MODIFIED / FOREIGN) yes/no 二次确认; 来自不同 artifact 的同名冲突 (DUPLICATE_NAME) 弹三选项: 覆盖原有 / 改用后缀名 / 取消.
  • Manifest .skillsjars-helper.json: 写入每个导出目录, 包含 artifact + skillRoot + 每文件 sha256, 用于判断升级与本地修改, 无需重型索引.
  • 安装徽标: skill 叶子节点 右对齐 在 skill name 同一行尾, 显示一组各 Agent 官方品牌图标 (Claude / Codex / Junie / Cursor / Gemini / Qoder, 自定义目录用文件夹图标); cell 宽度被强制撑到 tree 宽度, 不同节点的徽标列视觉对齐. 品牌资产取自 @lobehub/icons (MIT). tooltip 仍以 纯文本明确列出 agent id, 兜底图标识别度. 启动与每次导出后, InstallationRegistryService 自动重扫 7 个预设目录的 manifest.
  • 测试: 新增 ExportNamingTest / ManifestJsonTest / ExportPlannerTest, 完整覆盖 6 种安装状态; SkillJarParserTest 增补 文件枚举用例.

2026.1.1000

  • Discovery pipeline: scans Maven regular dependencies and Maven plugin <dependencies>, parses META-INF/skills/**/SKILL.md and META-INF/resources/skills/**/SKILL.md inside JARs, and exposes results through a project-level SkillRegistry service.
  • Extension point: introduces dev.dong4j.idea.skillsjars.helper.skillSourceScanner so future Gradle / SBT scanners and third-party plugins can plug in without changes to the coordinator.
  • Tree-based Agent Skills tool window: artifacts are grouped one level deep, skills become leaves; a dedicated description panel sits below a draggable splitter and shows the full description of the selected skill.
  • Tree interactions: double-click or Enter opens the in-jar SKILL.md; right-click offers Open SKILL.md / Copy Skill Name / Copy Coordinate / Open Source Jar; type-to-filter speed search; hover tooltips; link-style empty state with inline Refresh.
  • Toolbar: minimal Refresh / Expand All / Collapse All; status bar shows the global summary only.
  • Internationalization: bundle messages updated for English and Simplified Chinese.
  • Tests: 25 unit tests covering coordinate parsing, SKILL.md frontmatter parsing, JAR-level parsing and the tree model.
